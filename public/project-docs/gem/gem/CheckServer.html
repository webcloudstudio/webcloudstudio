NAME

CheckServer.pl - Utility to check database servers

SYNOPSIS

This flexible program checks Sybase and MSSQL servers.

Things that are always monitored:

  Server Availablility
  "Zombie" processes (Sybase only)
  Blocked processes
  Processes that are in bad state ("infected", "log suspend", "bad", etc.)

These conditions are considered CRITICAL or ERROR. email alerts will be sent to both --MAILTO and --WARN_MAILTO recepients. 


Things that are monitored based on command line arguments:
  Long running transactions
  Processes with high CPU or I/O
  High number of runnable processes
  Blocked processes (and circular deadlocks)

These conditions are considered WARNINGs. eMails will be sent to --WARN_MAILTO recepients only.

USAGE

The program can operate in two modes:

Persistent mode

Persistent mode simply connects to all your servers and loops through them running a small, quick, query. The program then sleeps --SLEEPTIME seconds and continues. This will test whether your connection is broken, and if it is, will alert. To allow configuration via older perl modules that have number of connection limits, you can use command line arguments to cycle through your registered servers. For example, --FIRSTSERVER=21 --LASTSERVER=40 will loop through servers 21-40 within your password file.

CheckServer.pl -BATCH_ID=id --PERSIST [--SLEEPTIME=120] [--FIRSTSERVER=n --LASTSERVER=m] [--[NON]PRODUCTION] [--PERSIST_MAIL_DELAY=300] [--DOALL] -OUTFILE=Outfile --MAILTO=a,b,c --WARN_MAILTOTO=x,y,z ...

Regular mode

Regular mode (No --PERSIST) will attempt to connect to your server, check it, and alert if it detects a problem.

CheckServer.pl -BATCH_ID=id -USER=SA_USER -SERVER=SERVER_NAME -PASSWORD=SA_PASS -OUTFILE=Outfile --MAILTO=a,b,c --WARN_MAILTO=x,y,z ...

COMMAND LINE ARGUMENTS - STANDARD ARGUMENTS

--DEBUG

Instructs program to run with informational printout

--BATCH_ID=batch_ID

Unique identifie of this particular program - used to track alerts and runtimes

--NOLOCK

To avoid ``spam'', the program is not allowed to run against the same server until ``threshold timeout'' (120 sec) between the runs expires. --NOLOCK switch overrides this restriction

--OUTFILE=log_file_name

Specifies log file name

COMMAND LINE ARGUMENTS - SERVER SELECTION

The following arguments define which servers the program will run on:

--DOALL={sybase|sqlsvr}

Request to monitor either: --DOALL=sybase - Sybase database servers only --DOALL=sqlsvr - MSSQL database servers only

--FIRSTSERVER=number --LASTSERVER=number

Request to monitor only servers in range (x - y)

--[NON]PRODUCTION

Request to monitor only production (--PRODUCTION), or non -production (--NOPRODUCTION) servers

-or-

-PASSWORD=......

Password for Server login ID

-SERVER=server_name

Name of the server to monitor

-USER=login_ID

Login ID (with sa and sso roles) to be used to login to server

COMMAND LINE ARGUMENTS - BEHAVIOR

The following command line arguments can be specified:

--BLOCKTIME=30

Specifies blocked process time threshold (sec). Default is 30 sec.

--HIGHCPU=5000

Specifies high CPU utilization threshold. By default high CPU utilization is not monitored (--HIGHCPU=0)

--HIGHIO=50000

Specifies high I/O threshold By default high I/O is not monitored (--HIGHIO=0)

--LONGRUNNING=120

Specifies long running transaction threshold Default is 120 min (2 hours)

--MAILTO=a,b,c

Comma separated list of eMail recepients that will be notified in case of Critical or Error conditions only

--NOMONITOR

If specified, only server availability and ``bad'' statuses are being monitored.

--NOSA

Supresses checks for processes initiated by user with privileged roles: sa, sso, oper

--PERSIST

Specifies the persistent mode of operation

--PERSIST_MAIL_DELAY=300

(Persistent mode). Specifies mail delays in sec. (to avoid spam). The default is 300 sec

--RUNNABLE=xxx

Specifies runnable process count threshold as multiplier of the number of engines on the monitored server. By default runnable process count is not monitored (--RUNNABLE=0)

--SLEEPTIME=120

(Persistent mode). Specifies inactivity time between the consecutive runs. The default is 120 sec (--SLEEPTIME=120)

--WARN_MAILTO=x,y,z

Comma separated list of eMail recepients that will be notified in case of Critical, Error or Warning condition

--ZOMBIE=60

``zombie'' process time thresholdi (in minutes). Default is 60 min.

NOTES

To supress server monitoring, add NOT_MONITORED=1 clause to the server password file.
