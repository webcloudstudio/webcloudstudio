REQUIREMENTS

The tools require preinstallation of perl (5.004 or later) and perl's Sybase::Dblib module (sybperl) on your system. This is pretty easy to implement on your own.  Installation of perl on your PC is a 10 minute process and perl is commonly installed on unix systems today (ie. your Unix admin can easily do it for you if you dont know how).

It is also useful (not required) to install Apache (or some other web server) on the system on which you install this package. As with Perl, Apache can easily be downloaded and installed on your NT workstation.  I believe   that web pages provides the best way to format and deliver information   to you and your users.  This package includes Eds System Documenter, a tool that documents your servers using a set of static  html  reports that are refreshed at night. The package also includes Webmonitor II, a full featured browser based system for managing your servers.    Eds System Documenter requires that you have access to a web server somewhere either   locally or via ftp (it will copy new static reports each night).   Webmonitor II requires a web server local to the system on which you install this package.  

You also must have sa access to to your servers to install the extended stored procedure library.

INSTALLATION INSTRUCTIONS

Once you have the prerequisite packages installed, download the software in gz format (NT or UNIX) and install as follows:
  1. Extract the software: After you download the software, the resulting  file should have a .gz extension (latest.tar.gz). Some older browsers will  strip extensions incorrectly so rename the file if it is not named latest.tar.gz.   On NT, extract using winzip and on UNIX, extract it with gunzip latest.tar.gz     (or gzip -d latest.tar.gz) followed by tar xvf latest.tar.  It is recommended that on NT you *NOT* install   into the Program Files subdirectories. The extra space in the  directory name may cause problems in some circumstances (like where you pass a full path to a file to a command). I think there wont be a problem, but why take the risk. The extraction  process will create a subdirectory ADMIN_SCRIPTS.  Extract this software  into the directory of your choice.
  2. Copy and modify configure.cfg, sybase_passwords.dat , and unix_passwords.datfrom the sample files
  3. Edit configure.cfg based on the sample
    4. to create configuration information (what packages you wish to install etc...)
  4. Edit sybase_passwords.dat based on the sample.
    5. to define your sybase servers list.
  5. Edit unix_passwords.dat based on the sample.
    6. to define your host list.
  6. run configure.pl(note: on nt, you can just click on the icon from file manager)
  7. Install cron entries (on unix) or schedule entries using your NT scheduler.  This can be based on crontab.sample which shows typical entries for a 4 server environment.
