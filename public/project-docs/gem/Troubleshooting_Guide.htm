Sql Server is not pingable 

Example:  system=ADSRV197 subsystem=ping state=CRITICAL Win32Ping Ping from ADSVM092 to ADSRV197 Failed at Tue Apr 26 10:42:34 2011 and Tue Apr 26 10:43:04 2011

Message Generated By Win32Ping Sent By MlpAlarmRoutingAgent.pl at Tue Apr 26 10:43:04 2011 From Host sybmon 

Procedures:
•              Ping the box and see if the server is down.
•              If the server is down, need to involve wininfra to start the server.
•              If the server is online, run osql or sqlcmd to connect to the sqlserver instance, make sure if the service is down.
•              If the service is down, bring it online and check the sqlserver error log to find the root cause.
•              If the service is not down, run sp__uptime to make sure if the service was restarted just now and check the error log.

Maintenace mail 

Example:  ADSRV141 Daily Maintenance Succeeded
Procedures:
•              Read the mail to make sure the maintenance plan run successfully.
•              If not, check the job history and dumpreports to find out why it failed and fix the problem.

Sybase blocking alerts 

Example:  ========== SSDB Summary Information ==========    
Blocked Process(es) found: 1
Longest Block Time:  7 Minutes 57 Secs       
========== Process Summary Information ==========
 
Procedures:
•              If this is because of dumping a database, can be ignored.
•              If not, login the server and run sp__whodo to find out which session is blocking others. If this blocking keeps too long (> 30 mins) and other jobs are blocked, need app owner to be involved and call Ed.

Sybase server is down. 

Procedures:
•              Run isql to make sure if the server is down. 
•              Ssh to the server with sybase@sybmon, checked the error log file (probably stack overflow).
•              Call Ed if needed, get approval to restart the service.

Non-native log shipping issue 

Example:  system=DO_ALL_LOOP subsystem=BROKERSQL3 state=CRITICAL CRITICAL: do_all_loop.ksh BROKERSQL3 on adsrv004 may be stalled 
Message Generated By HEARTBEAT Sent By MlpAlarmRoutingAgent.pl at Mon Apr 25 22:54:23 2011 From Host sybmon 
Procedures:
•              Remote to ADSVM092 to check log shipping window to see which database is being restored.
•              Go to logshipbuffer directory to check if the tran log file is too large. May apply a full dump to save time.
•              If there are databases out of sync, copy the latest full dumps to DR and load them.

Incomplete SQL Server Messages

  • Source Monitor Win32Eventlog
  • What: Event log messages are local to the monitoring box, some messages cant be shown b/c they don’t exist on local box
  • Example Mail Subject: E.MUREXDW MSSQLSERVER %1
  • Example Mail Body: system=MUREXDW subsystem=MSSQLSERVER state=ERROR %1
  • Followup Procedures: Look at the event log on the box for sql server errors
  • Permanent Solutions: edit ~gem/ADMIN_SCRIPTS/bin/win32_eventlog.messagedata and add line with particular message id so it can translate.

Double Monitoring

  • Source Monitor Any
  • What: You get two messages from the same system
  • Permanent Solutions: Make sure the system is only in the config files once
