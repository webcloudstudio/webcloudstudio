The Commander . Your role changed: not developer/designer to Commander (the Agile product owner). Act 3 . Walk the phases (3:50–9:00)

S / Set up . pipx install, config, init. Runs on Claude/Codex subscription. No API keys. Just a subscription and an idea.

A / Analyze . Import md/source/Spec Kit/notes to analyze decomposes into stories, milestones, blockers, questions. Blocker to stops and asks. Output = a tentative set of Features, Stories, Spikes; nothing committed until you approve.

QuarterDeck (core differentiator) . Your Agile web interface to the process: an optimized path between Commander and the L L M team. Renders the story hierarchy, blockers, questionnaires; approve/answer/redirect to persisted to carried forward. rigging compact gives minimal context with Builder/User views. Instead of guessing, the path 25 years of agile already worked out. Commander controls to goal is working software you can iterate.

Manifest (engine) . plan to typed Blueprints plus dependency graph. L L M estimates Story Points (token cost) per story against the graph. Drydock stacks the minimal correct files into each build prompt. Context engineered, not hoped for.

I / Implement . build walks the graph, builds the runnable frontier one context-optimized step at a time; evidence per step; verify against acceptance to unlock dependents. build score = 7-dimension delivery health (completeness, tests, drift…).

Rigging . Branding plus stack rules injected everywhere. Builder gets full spec; consumer gets compacted how-to. Keeps context lean at scale.

L / Loop . refit, two ways: (1) edit spec files to tracks git commit per file to rebuilds only what changed; (2) change tickets against existing specs, built with minimized context. Spec stays source of truth. Minimal drift.
